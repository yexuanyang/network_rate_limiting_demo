# 网络下载速度限流 Demo

一个使用 C++ 实现的 TCP 文件下载服务，内置**双层令牌桶算法**进行速率限制，支持每连接独立限速和全局聚合限速。

## 功能特点

- **双层令牌桶限流**：同时支持单连接限速和全局聚合限速
- **连接数控制**：限制最大并发连接数，超限时拒绝新连接
- **实时进度显示**：客户端实时显示下载进度、速度和限速配置
- **动态速率调整**：运行中可通过命令行动态调整客户端限速
- **SO_REUSEPORT 支持**：多进程共享同一端口，内核分配连接
- **网卡绑定**：支持绑定到特定网络接口
- **灵活配置**：丰富的命令行参数支持

## 构建

```bash
mkdir build && cd build
cmake ..
make
```

## 使用方法

### 1. 启动服务器

```bash
# 默认配置 (端口 8080, 不限速, 当前目录)
./server

# 自定义配置
./server --port 9000 --global-rate 102400 --per-conn-rate 51200 --root /path/to/files
```

#### 服务器参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--port` | 监听端口 | 8080 |
| `--root` | 文件根目录 | 当前目录 |
| `--global-rate` | 全局聚合限速（字节/秒），0 表示不限 | 0（不限速） |
| `--per-conn-rate` | 单连接限速（字节/秒） | 不限速 |
| `--max-clients` | 最大并发连接数 | 64 |
| `--iface` | 绑定到特定网卡（如 eth0） | 任意网卡 |
| `--reuseport` | 启用 SO_REUSEPORT，允许多实例共享端口 | 关闭 |

#### 示例

```bash
# 启动服务器，全局限速 100 KB/s，单连接限速 50 KB/s
./server --global-rate 102400 --per-conn-rate 51200

# 启动 2 个实例共享端口 9000（需要 SO_REUSEPORT）
./server --port 9000 --reuseport &
./server --port 9000 --reuseport &
```

### 2. 下载文件

```bash
# 下载文件
./client --file example.txt

# 指定服务器地址和输出文件名，客户端本地限速 200 KB/s
./client --file example.txt --host 192.168.1.100 --port 9000 --output my_file.txt --rate 204800
```

#### 客户端参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--file` | 要下载的文件名（必需） | - |
| `--host` | 服务器地址 | 127.0.0.1 |
| `--port` | 服务器端口 | 8080 |
| `--output` | 输出文件名 | downloaded_\<文件名\> |
| `--rate` | 客户端本地限速（字节/秒） | 100 KB/s |

#### 运行时命令

客户端启动后，支持以下交互命令：

- `rate <bytes_per_sec>`：动态调整客户端限速，如 `rate 51200`（50 KB/s）
- `quit` / `exit`：退出下载

### 3. 运行测试

```bash
cd build
./test_server
```

测试包含 5 个用例：

| 测试 | 说明 |
|------|------|
| T1 | 文件完整性验证（4 MB 文件） |
| T2 | 单连接限速验证（8 MB @ 2 MB/s） |
| T3 | 全局聚合限速验证（3 客户端 × 6 MB，限速 3 MB/s） |
| T4 | 连接数上限控制（max=2，验证 SERVER_BUSY） |
| T5 | SO_REUSEPORT 多实例共享（2 实例，4 客户端） |

也可以使用 `ctest` 运行：

```bash
cd build
ctest --output-on-failure
```

│              │  TCP    │                                 │

│  接收数据    │◄────────│  单连接令牌桶限流               │

|------|------|
| `include/rate_limiter.h` | 令牌桶限流器实现（线程安全） |

## 限流算法

### 令牌桶算法 (Token Bucket)

```
                    ┌─────────────────┐
    令牌生成 ──────►│     令牌桶       │─────► 发送数据消耗令牌
   (rate/sec)      │   (容量=rate)    │      (1字节=1令牌)
                    └─────────────────┘
                         ▲
                         │
                   令牌不足时阻塞等待
```

**工作原理**：
1. 桶中以固定速率生成令牌（`bytes_per_second` 个/秒）
2. 发送数据需要消耗对应字节数的令牌
3. 令牌不足时线程阻塞等待，直到令牌充足
4. 桶有容量上限（`max_tokens_`），允许一定程度的突发传输

### 双层限速

服务端支持两层限速机制：

1. **单连接限速**（`per-conn-rate`）：每个客户端连接独立的令牌桶
2. **全局聚合限速**（`global-rate`）：所有连接共享的令牌桶

```
Client 1 ───► Per-conn Bucket 1 ────┐
                                    │
Client 2 ───► Per-conn Bucket 2 ────┼──► Global Bucket ──► Network
                                    │
Client 3 ───► Per-conn Bucket 3 ────┘
```

数据发送时需要同时通过两个令牌桶，确保：
- 单连接不超过配置的速率
- 所有连接的总吞吐量不超过全局限制

## 协议说明

客户端请求格式：`@filename\n`

- 以 `@` 字符开头，后跟文件名，以换行符结束
- 服务端响应：文件内容流，或错误信息（`FILE_NOT_FOUND` / `SERVER_BUSY`）
